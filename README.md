# 23 — 와인 정복 게임 (VINUM IMPERIUM)

와인 지식 퀴즈 × 경영 시뮬레이션 브라우저 게임. `index.html` 더블클릭으로 실행 (설치·서버 불필요).

## 컨셉

로마 또는 조지아 진영을 골라, 포도씨를 들고 고대 세계 10개 지역을 정복한다.
동물의 숲(계절 턴제·아늑함) × 거상(시세·카라반 교역) × 페르시아의 왕자(시간의 모래 퀴즈 타이머).

## v2 — 지도 비주얼 + 캐릭터

- **SVG 고지도**: 지중해~페르시아 양피지풍 지도. 정복 시 진영 깃발 팝 애니메이션 + 영토 글로우, 붉은 펄스 링 = 정복 가능, 자물쇠 = 봉인
- **캐릭터 생성**: 이름 입력 + 초상 선택. 진영별 전통복장 여5/남5 (로마: 스톨라·팔라/토가·월계관, 조지아: 카르툴리 드레스·레차키/초하·파파키) — 전부 인라인 SVG
- **캐릭터 마커**: 지도 위에서 원정·정복에 따라 이동 (이동 애니메이션)
- 구버전 세이브 호환: 이어하기 시 캐릭터 생성만 추가로 진행

## 핵심 루프

1. **개간** — 포도씨 + 골드로 포도밭 조성 (지역당 최대 3개)
2. **재배** — 「다음 계절」로 시간 경과, 봄~가을 성장 (겨울 휴면)
3. **수확 → 양조** — 포도 2 → 와인 1 (양조장 필요)
4. **교역** — 지역별 변동 시세로 판매, 카라반 원거리 교역 (2배 값, 떼일 위험 20%)
5. **원정** — 물자+와인을 들고 가서 **와인 퀴즈 대결** 5문제 중 3문제 정답 시 정복

## 퀴즈

- **첫 정복 튜토리얼**: 새 게임의 첫 정복 1회는 초초급 3문항 + 학습모드(시간제한 없음, 오답 시 해설 보고 재도전 → 패배 불가, 규칙 즉석 코칭). 완료 후 정식 규칙으로 전환 (`S.tutorialDone`으로 1회성, 기존 세이브는 미발동)
- 지역 티어 = 난이도: WSET L1 → L2 → L3 → Diploma → **Master of Wine** (5단계 × 30문항 = 150문항, 해설 포함)
- 문항은 양조·재배 뉘앙스, 산지·등급·생산자·빈티지 구체성, 실무·유통 지식을 아우르는 정교한 세트 (전 티어 상향)
- **중복 방지**: 티어별로 이미 낸 문항은 그 티어를 전부 소진하기 전까지 다시 나오지 않음 (`S.usedQ`에 기록, 세이브에 유지)
- **보기 셔플**: 출제 때마다 보기 순서를 섞어 정답 위치가 고정되지 않음 (`shuffleQ`)
- **정답 확인하기**: 모르는 문제를 물자(티어×4)로 넘길 수 있음
- 30초 모래시계 타이머 — 시간 초과 시 오답
- 문항 정합성 검증: `node tools/validate-quiz.mjs` (정답 인덱스·보기 개수·중복 자동 점검)

## 이벤트 (계절마다 확률)

부정과 긍정 이벤트를 가중치로 뽑되, **긍정이 부정보다 1.15배 더 자주** 발생하도록 설계 (난이도별 `evtNeg`가 부정 총확률, 긍정 = ×1.15). 지진은 종류 다양화로 비중이 크게 낮아짐.

- **부정**: 지진(포도밭/건물 붕괴) · 야만족 침공(퀴즈 방어전, 성벽으로 예방) · 가뭄(성장 후퇴) · 우박(포도밭 결딴) · 파업(물자/포도 손실) · 대금 떼임
- **긍정**: 선호층 확보(+골드) · 와인값 상승(이번 계절 전 시장 시세↑) · 투자자 확보(+골드) · 풍년(전 포도밭 성장) · 상인 특가(판매 2배/씨앗 반값) · 음유시인(물자 +5)

## 기타

- 음악: WebAudio 칩튠 루프 (E단조 8마디), 헤더에서 토글
- 자동 저장: localStorage (`vinum_save`) — 브라우저 껐다 켜도 이어하기
- 승리: 10개 지역 전부 정복

## 명예의 전당 백엔드

전역 전당은 Google Apps Script 웹앱으로 동작한다. 웹앱 URL은 공개되므로 **누구나 POST 가능**하다고 가정하고, 클라이언트(`index.html`)와 서버(`hall-backend.gs`) 양쪽에서 입력을 검증·정제한다.

- 클라이언트: 전송·저장 전 `sanitizeHallEntry`로 제어문자 제거·길이 제한, 승리 상태에서만 등록
- 서버(`hall-backend.gs`): 필드 화이트리스트, HTML/제어문자 제거, 값 범위 강제, 간단한 레이트리밋
- **주의**: 서버 검증을 실제로 적용하려면 `hall-backend.gs`를 Apps Script에 붙여넣고 **새 버전으로 재배포**해야 함 (파일 안 배포 안내 참고)

## 파일

- `index.html` — 게임 전체 (단일 파일, 의존성 없음)
- `hall-backend.gs` — 명예의 전당 서버 코드 (Apps Script 배포용)
- `tools/validate-quiz.mjs` — 퀴즈 데이터 검증 스크립트
